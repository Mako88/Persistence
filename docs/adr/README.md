# Architecture Decision Records

Short records of decisions that shape the codebase — *what* was decided, *why*, and *what was
rejected* — so the rationale survives and the decision isn't silently re-litigated or violated.

Generic patterns (the cross-project baseline) live in global memory and `docs/CONVENTIONS.md`; these
ADRs record decisions made **for Persistence**, including where we deliberately apply (or deviate
from) a generic pattern.

Format: Status / Context / Decision / Alternatives considered / Consequences. Keep them short. Add a
new numbered file per decision; mark superseded ones rather than deleting.
